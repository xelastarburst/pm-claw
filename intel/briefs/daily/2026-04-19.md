## 🟠 Crashes & Stability

### 1. Isaac Sim 5.1 crashes on RTX 5060 Ti Blackwell — rtx.scenedb.plugin access violation
- **URL:** https://forums.developer.nvidia.com/t/isaac-sim-5-1-crashes-on-startup-with-rtx-5060-ti-blackwell-sm-120-rtx-scenedb-plugin-crash/366252
- **Version:** Isaac Sim 5.1.0-rc.19 | Windows 11 25H2 | RTX 5060 Ti (sm_120) | Driver 595.79
- **Category:** crashes-stability
- **Severity:** pain
- **Source:** nvidia-forum | 2026-04-11
- **Summary:** Fifth confirmed Blackwell GPU crash. Immediate access violation in rtx.scenedb.plugin during carbOnPluginStartup. NVIDIA moderator confirmed this is a known issue with the 595.xx driver branch on Blackwell. Downgrade to 591.74 resolves — same fix as 5070 Ti case.
- **Signal:** Blackwell-595 driver compatibility is now a confirmed, reproducible pattern. 5060 Ti joins 5070 Ti, 5080, 5090, and PRO 6000 in the broken-Blackwell list. Validated driver (580.88) remains 15+ versions behind current Windows default.

### 2. Isaac Sim 5.1.0 Windows crash on RTX 6000 & RTX A4000 — renderer plugin failure
- **URL:** https://forums.developer.nvidia.com/t/isaac-sim-5-1-0-crash-on-windows/366768
- **Version:** Isaac Sim 5.1.0 | Windows | RTX 6000 + driver 595.79 / RTX A4000 + driver 582.08
- **Category:** crashes-stability
- **Severity:** pain
- **Source:** nvidia-forum | 2026-04-16
- **Summary:** Two-system report. RTX 6000 crashes after startup; RTX A4000 crashes during MAVLink handshake / ZMQ. Backtrace: gpu.foundation.plugin!omni::activity::getInstance → omni.kit.renderer.plugin. Moderator confirms driver 595.79 R590 branch unvalidated; recommended 580.88.
- **Signal:** Blackwell-595 issue is widening beyond consumer RTX 50-series into professional RTX 6000 workstations. Windows auto-update is silently pushing users onto broken drivers.
