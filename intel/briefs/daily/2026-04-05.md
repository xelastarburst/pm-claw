## 🔴 Crashes & Stability

### 1. Isaac Sim 5.1 GUI Crash on RTX 5070 Ti (Blackwell) — Fixed by Driver Downgrade (5.1.0)
- **URL:** https://forums.developer.nvidia.com/t/isaac-sim-5-1-gui-crash-access-violation-on-rtx-5070-ti-blackwell-fixed-by-driver-downgrade-to-591-74/365335
- **Version:** Isaac Sim 5.1.0 | Windows 11 | RTX 5070 Ti Laptop (12GB)
- **Category:** crashes-stability
- **Severity:** pain
- **Source:** NVIDIA Forums | 2026-04-01
- **Summary:** Isaac Sim 5.1.0 crashes ~20 seconds after GUI startup with "access violation in omni.kit.window.preferences" on RTX 5070 Ti with driver 595.79. Headless mode works fine. Downgrading to driver 591.74 resolves the issue. NVIDIA confirmed validated driver is 580.88 and flagged Vulkan rendering path as known internal concern for Blackwell.
- **Signal:** Third Blackwell GPU crash report (after RTX 5080 segfault, RTX 5090 TiledCamera hang). Blackwell compatibility remains a systemic gap.

### 2. Isaac Sim Crash with NVIDIA Driver 595.71 on Blackwell GPUs (5.1.0)
- **URL:** https://github.com/isaac-sim/IsaacSim/issues/517
- **Version:** Isaac Sim 5.1.0 | Windows 11 | RTX 5070 Ti
- **Category:** crashes-stability
- **Severity:** pain
- **Source:** GitHub | 2026-03-03
- **Summary:** Isaac Sim 5.1.0 crashes immediately on launch with driver 595.71. Multiple users confirmed same issue with 595.71 and 595.79. Official recommendation: use validated driver 580.88 (Windows) or 580.65.06 (Linux). Also affects multi-GPU setups with Blackwell GPUs. Still OPEN.
- **Signal:** Converging with forum reports — Blackwell + 595.x drivers = broken Isaac Sim. Users upgrading drivers for other reasons get burned.

## 🟠 Sensors & Synthetic Data

### 3. Replicator Annotators Give Empty Frames on Multi-GPU (5.1.0) — FIXED
- **URL:** https://github.com/isaac-sim/IsaacSim/issues/507
- **Version:** Isaac Sim 5.1.0 | Ubuntu 24.04 | 2x RTX 6000 Ada
- **Category:** synthetic-data
- **Severity:** pain
- **Source:** GitHub | 2026-02-26 (CLOSED 2026-04-01)
- **Summary:** Semantic segmentation annotators randomly produce fully transparent/empty frames when using multiple GPUs (~50% reproducibility). RGB images from same frames are normal. - **Signal:** Multi-GPU synth data generation is critical for production SDG workflows. Fix landing is positive.

### 5. ROS 2 Jazzy Bridge Python Version Mismatch with Isaac Sim 5.1 (5.1.0)
- **URL:** https://forums.developer.nvidia.com/t/isaac-sim-tutorial-step-6-difficulty-setting-up-ros-2-jazzy-bridge-with-isaac-sim-5-1-on-ubuntu-24-04-python-version-mismatch/362199

- **Category:** integration
- **Severity:** pain
- **Source:** NVIDIA Forums | 2026-03-02
- **Summary:** Official tutorial Docker build produces Python 3.12 rclpy binaries, but Isaac Sim requires Python 3.11. Results in "No module named 'rclpy._rclpy_pybind11'" error. Second user confirmed same issue on March 23. This is a different ROS 2 incompatibility than the IGX Orin kernel conflict.
- **Signal:** Second distinct ROS 2 + Isaac Sim incompatibility. Pattern: Python version ecosystem fragmentation between ROS 2 and Isaac Sim.
